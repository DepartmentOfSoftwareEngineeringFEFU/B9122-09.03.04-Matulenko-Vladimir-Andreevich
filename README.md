# Визуализация природных явления в 3D-картах

Дипломный проект по теме: "Визуализация природных явления в 3D картах"

## Технологический стек

| Компонент | Технологии |
|-----------|-----------|
| **Backend** | Python, FastAPI, SQLAlchemy (SQLite), aiohttp |
| **Frontend** | React, Three.js (@react-three/fiber), Tailwind CSS |
| **Сборщик** | Vite |

## Быстрый старт

### Требования

- **Python** 3.10+
- **Node.js** 18+
- **npm** (устанавливается вместе с Node.js)

### 1. Клонирование репозитория

```bash
git clone https://github.com/ВАШ_ЛОГИН/Diplom.git
cd Diplom
```

### 2. Настройка Backend

```bash
# Переходим в папку бэкенда
cd backend

# Создаём виртуальное окружение
python -m venv .venv

# Активируем его (Windows PowerShell)
.\.venv\Scripts\activate
# Для Linux/macOS: source .venv/bin/activate

# Устанавливаем зависимости
pip install -r requirements.txt
```

### 3. Настройка Frontend

```bash
# Открываем новый терминал
cd frontend
npm install
```

### 4. Запуск (нужны два терминала)

**Терминал 1 — Backend (FastAPI):**
```bash
cd backend
# Убедитесь, что виртуальное окружение активировано
uvicorn main:app --reload
```
Сервер запустится на `http://127.0.0.1:8000`

**Терминал 2 — Frontend (React):**
```bash
cd frontend
npm run dev
```
Приложение откроется на `http://localhost:5173`

### 5. Использование

1. Откройте `http://localhost:5173` в браузере. По умолчанию приложение откроется на координатах г. Владивосток и загрузит актуальный ветер с Open-Meteo.
2. Для изменения локации загрузите JSON-файл симуляции (содержащий координаты, параметры тумана, землетрясений и кастомного ветра) через кнопку **«Метеоданные (JSON)»** в панели слева.
   - *Примеры файлов находятся в папке `tests/` (например, `tests/exp_extreme_weather.json`).*
3. После загрузки JSON, бэкенд автоматически скачает нужный тайл рельефа (AWS S3 Terrarium) по указанным координатам и передаст его на клиент.
4. Управляйте видимостью слоёв (рельеф, туман, ветер, землетрясения) через чекбоксы.
5. Настройте вертикальный масштаб (Z-Scale) для лучшей видимости перепадов высот.
6. Вращайте 3D-сцену мышкой (ЛКМ), перемещайте (ПКМ), масштабируйте колёсиком.

## Формат метеоданных (JSON)

Приложение позволяет загружать пользовательские сценарии погоды в формате JSON. Структура файла включает метаданные (для загрузки нужного участка рельефа) и параметры слоёв (землетрясения, туман, ветер).

Пример структуры JSON-файла:
```json
{
  "metadata": {
    "center_lat": 43.05,
    "center_lon": 131.89,
    "zoom": 11
  },
  "fog": {
    "density": 0.85,
    "top_height_m": 150.0,
    "color": "#e0e6ed"
  },
  "wind_stations": [
    {
      "name": "Станция Токаревский маяк",
      "lat": 43.07,
      "lon": 131.83,
      "azimuth_deg": 135.0,
      "speed_ms": 15.0,
      "color": "#ff6b6b"
    }
  ],
  "earthquakes": [
    {
      "magnitude": 5.5,
      "depth_km": 12.0,
      "lat": 43.10,
      "lon": 131.90
    }
  ]
}
```

### Описание полей

- **`metadata`** (Обязательно): Привязка карты.
  - `center_lat` / `center_lon`: Координаты центрального участка рельефа.
  - `zoom`: Масштаб карты (рекомендуется `10`-`12`).
- **`fog`** (Опционально): Параметры тумана.
  - `density`: Плотность от `0.0` до `1.0`.
  - `top_height_m`: Верхняя граница тумана (в метрах).
  - `color`: Цвет в HEX формате (например, `"#e0e6ed"`).
- **`wind_stations`** (Опционально): Массив метеостанций для кастомного ветра (IDW-интерполяция). Если не указано, данные скачиваются онлайн с Open-Meteo.
  - `name`: Название станции.
  - `lat` / `lon`: Координаты станции (должны быть в пределах карты для видимости).
  - `azimuth_deg`: Направление ветра (`0`-`360`). `0` — северный ветер (дует на юг).
  - `speed_ms`: Скорость ветра в м/с.
  - `color`: Цвет потоков станции в HEX формате.
- **`earthquakes`** (Опционально): Массив сейсмических активностей.
  - `magnitude`: Магнитуда (`1`-`10`).
  - `depth_km`: Глубина в км.
  - `lat` / `lon`: Точка эпицентра.
